Data transformation

To get a better view of the NYPD shooting data set I transform the following variables to factors:

  • INCIDENT_KEY
  • PRECINCT
  • JURISDICTION_CODE
  • BORO
  • LOCATION_DESC
  • STATISTICAL_MURDER_FLAG
  • PERP_AGE_GROUP
  • PERP_SEX
  • PERP_RACE
  • VIC_AGE_GROUP
  • VIC_SEX
  • VIC_RACE

After that I remove the variables I dont need:

  • X_COORD_CD
  • Y_COORD_CD
  • Latitude
  • Longitude
  • Lon_Lat

The final step in this first phase is transforming the variables containing a date (OCCUR_DATE) and time (OCCUR_TIME) into actual date and time variables.

# transform variables with data type character to factors
nypd_shooting_transform <- nypd_shooting %>% 
  mutate(INCIDENT_KEY = as.factor(INCIDENT_KEY),
         PRECINCT = as.factor(PRECINCT),
         JURISDICTION_CODE = as.factor(JURISDICTION_CODE),
         BORO = as.factor(BORO), 
         LOCATION_DESC = as.factor(LOCATION_DESC),
         STATISTICAL_MURDER_FLAG = as.factor(STATISTICAL_MURDER_FLAG),
         PERP_AGE_GROUP = as.factor(PERP_AGE_GROUP), 
         PERP_SEX = as.factor(PERP_SEX),
         PERP_RACE = as.factor(PERP_RACE), 
         VIC_AGE_GROUP = as.factor(VIC_AGE_GROUP),
         VIC_SEX = as.factor(VIC_SEX), 
         VIC_RACE = as.factor(VIC_RACE))

# Remove variables I don't need
nypd_shooting_transform_I <- nypd_shooting_transform %>% 
  select(-c(X_COORD_CD, Y_COORD_CD, Latitude, Longitude, Lon_Lat))

# Transform variables containing date and time into actual date and time variables
nypd_shooting_transform_II <- nypd_shooting_transform_I %>% 
  mutate(OCCUR_DATE = mdy(OCCUR_DATE),
         OCCUR_TIME = hms(OCCUR_TIME))

To see whether incident keys that appear multiple times affect the number of cases, I decided to plot the decrease in entries before and after removing the duplicate cases. I did this per borough, so I could see whether the number of entries for certain boroughs were affected disproportionately.

# Select all the incident keys in the NYPD Shooting data set
incident_keys <- nypd_recode_IV %>%
  select(INCIDENT_KEY) %>%
  group_by(INCIDENT_KEY) %>%
  summarise(Count = n()) %>%
  select(INCIDENT_KEY)

# Count all the distinct incident keys in the NYPD Shooting data set (grouped by borough)
incident_key_single <- incident_keys %>%
  left_join(nypd_recode_IV) %>%
  select(INCIDENT_KEY, BORO) %>%
  distinct(INCIDENT_KEY, BORO) %>%
  group_by(BORO) %>%
  summarise(Count = n()) %>%
  mutate(category = "single")

# Count all incident keys in the NYPD Shooting data set (grouped by borough)
incident_key_all <- nypd_recode_IV %>%
  select(BORO) %>%
  group_by(BORO) %>%
  summarise(Count = n()) %>%
  mutate(category = "all")

# Compute the difference between the number of incident keys in the original data set
# and the data set where every incident key appears only once. This is done per borough.
incident_key_total <- incident_key_all %>%
  inner_join(incident_key_single, by = "BORO") %>%
  rename(count_all = Count.x, count_single = Count.y) %>%
  select(BORO, count_all, count_single) %>%
  mutate(difference =  ((count_all - count_single)/ count_single))

# Plot that shows the decrease in observations when all incident keys 
# which appear more than once have been removed. This is done per borough.
plot_analysis <- incident_key_total %>%
  ggplot(aes(x= reorder(BORO,-difference), y = difference)) +
  geom_bar(stat= "identity", alpha=0.5) +
  scale_y_continuous(labels = scales::percent) +
  labs(x = 'Borough', y = 'Decrease', title = 'NYPD shooting: Difference original data set and data set without duplicates')

max_decrease <- round(max(incident_key_total$difference) * 100, 2)

min_decrease <- round(min(incident_key_total$difference) * 100, 2)

plot_analysis


What is interesting is that the decrease in entries is about the same for all five boroughs. Of course, the difference between the biggest (31.31%) and smallest (24.19%) decrease is not insignificant. However, even when removing all double entries across the five boroughs the original order remains intact:

# A plot of the original data set and the data set where all the duplicate identity keys have been removed

Conclusion

The analysis showed that an eventual removal of incident keys that appear more than once will not change which boroughs are featured most often. It does however, lead to additional questions. For instance, this analysis could also be applied to the precincts: Does removal of double incident keys lead to a change in which precincts appear the most in the NYPD shooting data set?

Also, the section Missing values is dedicated to recoding empty values to Unknown. It would be interesting to find out whether the amount of Unknown differs among the boroughs and precincts in the NYPD data set.

Bias

A possible source of bias could have been the final visualization in this document (NYPD shooting: Original data set and data set absent duplicates). It appears that after removing double incident keys the decrease in entries for Brooklyn and the Bronx is much higher than Queens, Manhattan and Staten Island. To me, it looked like Brooklyn and the Bronx were disadvantaged by the method of recording shooting incidents in the NYPD shooting data set.
However, the visualization right before that (NYPD shooting: Difference original data set deduplicated data set) shows that measured in percentages, the difference in entry decrease among the five boroughs is much smaller than it seems at first. Brooklyn, the borough with the largest decrease in absolute entries (1896) even has the smallest percentual decrease in entries in the NYPD shooting data set (24.19%).
So, plotting the relative changes for the five boroughs was the way to mitigate this personal bias.
